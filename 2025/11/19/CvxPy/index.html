❓ 问题引出

💭 问题：什么是 CvxPy？

CvxPy 是一个基于 Python 的 凸优化建模库。可以把它理解成：

  • 用类似数学公式的方式，在 Python 里写优化问题；
  • 把 变量 目标函数 约束条件 写清楚；
  • 然后交给 CvxPy 调用底层求解器 solver，自动算出最优解。

    • verbose=False：表示不输出求解器的详细日志信息。如果想查看求解过程中的日志，可以将其设置为 True。

    • gp=False：表示问题不是几何规划（Geometric Programming）。如果问题是几何规划，可以将其设置为 True。

    • qcp=False：表示问题不是二次锥规划（Quadratically Constrained Programming）。如果问题是二次锥规划，可以将其设置为 True。

    • requires_grad=False：表示不需要计算梯度信息。如果需要梯度信息，可以将其设置为 True。

    • enforce_dpp=False：表示不强制执行 DPP（Disciplined Parametrized Programming）规则。如果需要强制执行，可以将其设置为 True。

    • ignore_dpp=False：表示不忽略 DPP 规则。如果需要忽略，可以将其设置为 True。

    • **kwargs：keyword arguments，可以传递给求解器的其他参数，例如 max_iters、abstol、reltol 等，具体参数取决于所使用的求解器。

1.2 常见的求解器类型

常见开源求解器

  • OSQP

    • 问题类型：二次规划(QP)、二次目标 + 线性约束（凸二次）
    • 优点：对稀疏 QP 高效、支持 warm start、迭代法内存低、适合实时/滚动优化。
    • 可配置：eps_abs/eps_rel、max_iter、polish、warm_start。

  • SCS

    • 问题类型：广义锥问题（第一阶方法，适合大型问题近似解）
    • 优点：可处理大规模问题，内存占用低，收敛快但解粗糙（精度较低）。
    • 可配置：eps、max_iters、acceleration，支持并行（部分实现）。

  • ECOS

    • 问题类型：锥规划（SOCP、QCQP 可部分转换）
    • 优点：对 SOCP 和二次锥问题表现良好，速度快、可靠性高。
    • 可配置：max_iters、abstol/reltol 等。

  • CVXOPT

    • 问题类型：QP、LP、二次锥等（经典 Python 求解器）
    • 优点：实现成熟、支持多种问题，但对大型稀疏问题不如专用求解器效率高。
    • 可配置：多项参数及低级接口。

  • ECOS：基于内点法的二阶锥规划（SOCP）求解器，适合凸二次锥问题和线性锥问题。精度高，收敛快（迭代次数少），但对大规模稀疏问题内存/每次迭代成本较高。

  • SCS：基于一阶原语（交替方向乘子法/ADMM）的锥规划近似求解器，能处理更广泛的锥（包括大规模问题），但收敛为近似解，精度通常低于内点法。对非常大的问题和内存受限场景更友好。

  • OSQP：专门针对二次规划（QP/凸二次问题）的算子分裂二次规划求解器（ADMM 变体），对稀疏大规模 QP 非常高效且稳定，能提供较高精度且速度通常优于 SCS 在 QP 上的表现。

常见商业/高性能求解器（通常需要许可证）

  • Gurobi
    • 问题类型：LP、QP、QCP、MIP（混合整数）、二次锥等
    • 优点：工业级性能、鲁棒、支持并行、多种高级选项。
    • 注意：需商业授权（学术可免费申请）。cvxpy 有接口。
  • MOSEK
    • 问题类型：线性/二次/二次锥/锥/大类凸问题
    • 优点：对大型锥问题、二次锥问题表现优异，稳定且精度高。
    • 注意：商业授权/学术许可证。cvxpy 支持调用。
  • CPLEX
    • 问题类型：LP、QP、MIP 等
    • 优点：与 Gurobi 类似（IBM 产品），高性能但需授权。

其他可用求解器/接口

  • HiGHS

    • 问题类型：LP、MIP、一些 QP 支持（近年来进展）
    • 优点：开源，高性能，用于结构化线性问题。cvxpy 有间接接口或通过 scipy / OSQP 组件配合。

  • scs + gpu实现（如 SCS 的 GPU variant）

    • 适用于需要 GPU 加速的大规模问题（视实现与依赖）。

  • qpsolvers / quadprog（Python 包）

    • 问题类型：小型 QP 求解（教学/简单场景）。

如何在 cvxpy 中选择

  • 若是 QP（你目前的情况：二次目标 + 线性约束）：
    • 首选 OSQP（稀疏、支持 warm start）。若对精度或数值稳定性有更高要求，尝试 Gurobi 或 MOSEK（商业）。
  • 若是大型近似解／第一阶方法可接受：
    • SCS（大规模、快速但精度有限）。
  • 若问题含二次锥（SOCP）：
    • ECOS 或 MOSEK 通常更合适。
  • 若需混合整数（MIP）：
    • Gurobi、CPLEX、CBC（开源）等。

如何获取更多 solver 选项与诊断

  • 在 cvxpy 中，调用 problem.solve(solver=cp.SOLVER_NAME, verbose=True, **solver_options) 并打印 problem.solver_stats。不同 solver 支持的选项不同，参考各 solver 文档。
  • 若遇数值问题，试：
    • 改变 solver（ECOS、SCS、OSQP、MOSEK、Gurobi 等逐个尝试）；
    • 改善问题缩放/预处理；
    • 把部分硬约束改为软约束并在目标中惩罚。

在选择求解器时，需要注意以下几点：

  1. 问题类型：不同的求解器适用于不同类型的优化问题，例如线性规划、二次规划等。
  2. 性能：某些求解器在特定问题上可能表现更好，可以根据实际情况进行选择。
  3. 依赖库：某些求解器可能需要额外的依赖库，确保在使用前已正确安装。
